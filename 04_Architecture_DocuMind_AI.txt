Architecture OverviewDocuMind AI follows a classic three-tier architecture: a React single-page application (SPA) as the presentation layer, a FastAPI REST API as the business logic layer, and PostgreSQL as the data persistence layer. The Gemini API is integrated at the backend tier for all AI operations.2. High-Level Architecture2.1 System ComponentsComponentTechnologyResponsibilityFrontendReactUser interface, state management, API callsBackend APIFastAPIBusiness logic, AI orchestration, document generationDatabasePostgreSQLPersistent storage of all application dataAI EngineGemini APINLP analysis, question generation, report writingAuthJWTStateless session managementDoc Enginepython-docx / reportlab / python-pptx / openpyxlGenerate downloadable documents in multiple formatsChartsmatplotlibGenerate charts embedded in reports2.2 Architecture FlowUser interacts with React UI.React sends HTTP requests to FastAPI backend (with JWT in Authorization header).FastAPI validates JWT, routes request to appropriate handler.For AI operations: FastAPI calls Gemini API and processes response.For document generation: FastAPI uses python-docx / reportlab / pptx / openpyxl.FastAPI reads/writes data via SQLAlchemy ORM to PostgreSQL.FastAPI returns JSON response or file stream to React frontend.3.
